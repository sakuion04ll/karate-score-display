# 空手演武 演出用得点システム

大学の武道系部活動を盛り上げるイベント「ReLIVE BUDO」において、空手部の演武（組手）を演出するために開発した、Webベースの得点表示システムです。

## 制作背景：観客と演武をつなぐ「解説」の役割
「空手の組手は、ルールを知らないと何が起きているのか、どちらが勝っているのか分からない」
この課題を解決することが開発のスタートでした。イベント内での組手は、公式戦とは異なる独自のルール（1分間・6点先取）で行われるため、既存のスコアボードアプリでは対応できませんでした。そこで、観客が直感的に状況を理解でき、かつステージ上の演武を邪魔しないシステムを自作する必要がありました。

## 機能と、こだわったポイント
### 1. 舞台演出としての「デザインと視認性」
最も重視したのは、ステージ上の空手部員を引き立てるためのデザインです。スクリーンに投影した際、観客席のどこからでも数字がはっきり見えつつ、主役である演武の邪魔にならない配色やレイアウトにしました。私の役割として、コードのベースはAIに任せつつ、この「人の目にどう映るか」という微調整に多くの時間を割きました。

### 2. 本番のプレッシャーに負けない「操作性」
照明が暗い舞台袖での操作を想定し、絶対にミスが起きないUI/UXを設計しました。
- **キーボード操作:** マウスを使わず、特定のキーを押すだけで得点加算・タイマー操作が可能。
- **リカバリー機能:** 万が一の誤操作に備え、即座に取り消し（Undo）ができる機能を実装。
これらにより、オペレーターが手元を見ずに演武に集中できる環境を作りました。

### 3. AI活用による超短納期開発
システム開発にかけられる時間は1週間でした。この限られた期間で完成させるため、ロジックの記述には生成AIを積極的に活用しました。AIにコードを書かせ、人間（私）は仕様策定とUI/UXの最終調整に徹するという分業体制をとることで、短期間でバグのない安定したシステムを構築しました。

## 開発を通じて得た学びと今後の展望
今回の開発では、AIをコーディングのパートナーとして活用する実践的なスキルが身につきました。しかし同時に、**「どのような演出にしたいか」「オペレーターが焦らないUIは何か」といった、人間の感性や現場の文脈に関わる部分は、エンジニア自身が深く考え抜かなければならない**と痛感しました。

技術力（AI活用）と、ユーザー視点（デザイン・UX）の両輪があって初めて、実用的なプロダクトが生まれるという経験は、今後のエンジニア人生において大きな指針となりました。
